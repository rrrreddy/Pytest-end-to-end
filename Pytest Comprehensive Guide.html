Pytest Fixtures

Fixtures are functions that provide reusable test data, objects, or setup/teardown logic. They enable modular, scalable testing by eliminating code duplication.

Basic Fixture Definition

import pytest

@pytest.fixture
def sample_data():
    """Provides sample data for tests."""
    return {"name": "Alice", "age": 30, "email": "alice@example.com"}

def test_user_name(sample_data):
    assert sample_data["name"] == "Alice"

def test_user_age(sample_data):
    assert sample_data["age"] == 30

Fixture Scopes

Scopes determine how often a fixture is invoked and when teardown occurs. Choose the appropriate scope to optimize test performance and isolation.

function

Default scope. Fixture runs once per test function. Use for isolated, fast setup with no side effects.

class

Fixture runs once per test class. Useful for sharing state across test methods in the same class.

module

Fixture runs once per test module. Ideal for expensive setup operations needed by multiple test functions.

package

Fixture runs once per test package. Use for shared resources across multiple modules in a package.

session

Fixture runs once per entire test session. Best for very expensive operations like database connections.

Scope Examples

import pytest

# Function scope (default)
@pytest.fixture
def temp_file():
    file = open("temp.txt", "w")
    yield file
    file.close()

# Module scope - runs once per module
@pytest.fixture(scope="module")
def database_connection():
    conn = create_db_connection()
    yield conn
    conn.close()

# Session scope - runs once for entire test session
@pytest.fixture(scope="session")
def api_client():
    client = APIClient()
    client.authenticate()
    yield client
    client.logout()

Best Practice When to Use Each Scope

  • Function: Fast setup, no side effects, full isolation needed
  • Class: Sharing state across test methods, reducing setup overhead
  • Module: Expensive operations (database setup, API connections)
  • Session: Very expensive resources shared across all tests

Dynamic Scope

You can dynamically determine fixture scope based on command-line arguments or configuration.

def determine_scope(fixture_name, config):
    """Dynamically set scope based on CLI option."""
    if config.getoption("--keep-containers", None):
        return "session"
    return "function"

@pytest.fixture(scope=determine_scope)
def docker_container():
    container = spawn_container()
    yield container
    cleanup_container(container)

Fixture with Setup and Teardown

Use yield to separate setup from teardown logic.

@pytest.fixture
def database():
    # Setup
    db = Database()
    db.connect()
    db.create_tables()
    
    yield db  # Test runs here
    
    # Teardown
    db.drop_tables()
    db.disconnect()

Mocking & Patching with pytest-mock

Mocking replaces real objects with test doubles to isolate code under test from external dependencies.

Installing pytest-mock

pip install pytest-mock

Basic Mocking Example

import pytest
from myapp import send_email

def test_send_email(mocker):
    # Mock the email service
    mock_service = mocker.patch('myapp.email_service.send')
    
    # Call the function under test
    send_email("user@example.com", "Hello!")
    
    # Assert the mock was called correctly
    mock_service.assert_called_once_with(
        email="user@example.com",
        message="Hello!"
    )

Patching Functions

Use mocker.patch() to replace a function during a test. The patch is automatically removed after the test.

def test_api_call(mocker):
    # Mock requests.get to avoid real HTTP calls
    mock_response = mocker.Mock()
    mock_response.json.return_value = {"status": "success"}
    mock_response.status_code = 200
    
    mocker.patch('requests.get', return_value=mock_response)
    
    # Test your function that uses requests.get
    result = fetch_data_from_api()
    assert result["status"] == "success"

Patching Object Methods

class PaymentGateway:
    def charge(self, amount, card_token):
        # Real implementation calls external payment API
        pass

def test_process_payment(mocker):
    gateway = PaymentGateway()
    
    # Patch the charge method on this specific instance
    mock_charge = mocker.patch.object(gateway, 'charge', return_value=True)
    
    # Test payment processing
    result = gateway.charge(100.00, "token_123")
    
    assert result is True
    mock_charge.assert_called_once_with(100.00, "token_123")

Return Values & Side Effects

# Setting return value
mock_function = mocker.Mock(return_value=42)
assert mock_function() == 42

# Multiple return values
mock_function.side_effect = [1, 2, 3]
assert mock_function() == 1
assert mock_function() == 2
assert mock_function() == 3

# Raising exceptions
mock_function.side_effect = ValueError("Invalid input")
with pytest.raises(ValueError):
    mock_function()

Async Mocking

import pytest
from unittest.mock import AsyncMock

@pytest.mark.asyncio
async def test_async_fetch(mocker):
    # Mock async function
    mock_fetch = mocker.patch('myapp.fetch_data', new=AsyncMock())
    mock_fetch.return_value = {"data": "test"}
    
    result = await fetch_data()
    
    assert result["data"] == "test"
    mock_fetch.assert_awaited_once()

Assertion Methods

Method Description
assert_called() Assert mock was called at least once
assert_called_once() Assert mock was called exactly once
assert_called_with(*args, **kwargs) Assert last call was with specified arguments
assert_called_once_with(*args) Assert called once with specified arguments
assert_not_called() Assert mock was never called
assert_any_call(*args, **kwargs) Assert mock was called with these args at some point

Mocking Best Practices

1. Mock at System Boundaries

Mock external services, databases, APIs - not internal implementation details.

# Good: Mock at system boundary
def test_good_mock(mocker):
    mocker.patch("myapp.requests.get")  # External HTTP call
    
# Avoid: Mocking internal functions
def test_bad_mock(mocker):
    mocker.patch("myapp.utils._internal_helper")  # Internal detail

2. Patch Where Used, Not Where Defined

# If service.py imports: from myapp.utils import helper_function
# Patch where it's USED, not where it's defined
mocker.patch("myapp.service.helper_function")  # ✓ Correct
# NOT: mocker.patch("myapp.utils.helper_function")  # ✗ Wrong

Common Pitfalls

  • Mock not being called: Verify patch path matches import location
  • Mock returns Mock instead of value: Set return_value explicitly
  • Async mock not awaitable: Use AsyncMock instead of Mock
  • Wrong patch target: Patch where function is used, not defined

Integration Testing for Data Pipelines

Integration tests verify that different components of your system work together correctly with external systems like databases, APIs, and cloud services.

What to Test

  • Custom code interactions with external systems (S3, databases, APIs)
  • Schema validation and data transformations
  • Column ordering and data type preservation
  • Partition and configuration correctness
  • End-to-end pipeline functionality

Testing Strategy

Use containers for open-source systems (Spark, Iceberg) and mocks for proprietary services (AWS, Snowflake).

Docker Containers

Simulate databases, message queues, and services locally using Docker containers with testcontainers-python.

Mock External APIs

Use libraries like Moto (for AWS services) to mock external APIs without real infrastructure.

Integration Testing Best Practices

What to Test

  • Test that you're using integration features correctly, not the features themselves
  • Focus on interactions between custom code and external systems
  • Test schema validation, data transformations, and business logic
  • Prioritize tests for integrations whose breakage is costly to fix

What NOT to Test

  • Don't test whether create_table actually creates a table (trust the vendor)
  • Don't test third-party library functionality
  • Don't test every possible integration if low-value
  • Avoid overly complex fixtures that are hard to maintain

import pandas as pd

class CustomerTransformer:
    def transform(self, df):
        """Apply business transformations to customer data."""
        df = df.copy()
        
        df['full_name'] = df['first_name'] + ' ' + df['last_name']
        
        df['customer_tier'] = pd.cut(
            df['total_purchases'],
            bins=[0, 500, 1500, float('inf')],
            labels=['bronze', 'silver', 'gold']
        )
        
        df['has_email'] = df['email'].notna()
        
        df['days_since_signup'] = (
            pd.Timestamp.now() - pd.to_datetime(df['signup_date'])
        ).dt.days
        
        return df

def test_transform_creates_full_name():
    """Test full_name column creation."""
    input_df = pd.DataFrame({
        'first_name': ['John', 'Jane'],
        'last_name': ['Doe', 'Smith'],
        'email': ['john@example.com', 'jane@example.com'],
        'signup_date': ['2024-01-15', '2024-02-20'],
        'total_purchases': [1500.50, 2300.75],
        'status': ['active', 'active']
    })
    
    transformer = CustomerTransformer()
    result = transformer.transform(input_df)
    
    assert 'full_name' in result.columns
    assert result['full_name'].tolist() == ['John Doe', 'Jane Smith']

def test_transform_customer_tier():
    """Test customer tier categorization."""
    input_df = pd.DataFrame({
        'first_name': ['Alice', 'Bob', 'Charlie'],
        'last_name': ['A', 'B', 'C'],

Step 4: Data Quality Validators

# src/etl/validators.py
class DataValidator:
    def __init__(self, df):
        self.df = df
        self.errors = []
    
    def validate_schema(self, required_columns):
        """Validate required columns exist."""
        missing = set(required_columns) - set(self.df.columns)
        if missing:
            self.errors.append(f"Missing columns: {missing}")
        return len(missing) == 0
    
    def validate_no_nulls(self, columns):
        """Check for null values in critical columns."""
        for col in columns:
            if self.df[col].isnull().any():
                null_count = self.df[col].isnull().sum()
                self.errors.append(f"Column '{col}' has {null_count} null values")
        return len(self.errors) == 0
    
    def validate_row_count(self, min_rows=1):
        """Ensure minimum row count."""
        if len(self.df) < min_rows:
            self.errors.append(f"Expected at least {min_rows} rows, got {len(self.df)}")
            return False
        return True
    
    def validate_unique(self, column):
        """Check for duplicate values."""
        duplicates = self.df[column].duplicated().sum()
        if duplicates > 0:
            self.errors.append(f"Column '{column}' has {duplicates} duplicates")
            return False
        return True

def test_validator_schema():
    """Test schema validation."""
    df = pd.DataFrame({'a': [1], 'b': [2]})
    validator = DataValidator(df)
    
    assert validator.validate_schema(['a', 'b']) == True
    assert validator.validate_schema(['a', 'c']) == False
    assert 'Missing columns' in validator.errors[0]

def test_validator_nulls():
    """Test null validation."""
    df = pd.DataFrame({'id': [1, 2, None], 'name': ['A', 'B', 'C']})
    validator = DataValidator(df)
    
    assert validator.validate_no_nulls(['id']) == False
    assert validator.validate_no_nulls(['name']) == True

import pytest
from etl.extractors import PostgresExtractor
from etl.transformers import CustomerTransformer
from etl.validators import DataValidator

def test_complete_etl_pipeline(source_db_connection, setup_source_data, s3_mock):
    """Test complete ETL pipeline from extraction to validation."""
    
    extractor = PostgresExtractor(source_db_connection)
    raw_data = extractor.extract_customers()
    
    assert len(raw_data) > 0, "No data extracted"
    print(f"✓ Extracted {len(raw_data)} records")
    
    transformer = CustomerTransformer()
    transformed_data = transformer.transform(raw_data)
    
    assert 'full_name' in transformed_data.columns
    assert 'customer_tier' in transformed_data.columns
    assert 'has_email' in transformed_data.columns
    print(f"✓ Transformed data with {len(transformed_data.columns)} columns")
    
    validator = DataValidator(transformed_data)
    
    required_cols = ['customer_id', 'full_name', 'email', 
                     'customer_tier', 'has_email']
    assert validator.validate_schema(required_cols)
    assert validator.validate_row_count(min_rows=1)
    assert validator.validate_unique('customer_id')
    
    print(f"✓ Data quality checks passed")
    
    csv_buffer = transformed_data.to_csv(index=False)
    s3_mock.put_object(
        Bucket='etl-staging-bucket',
        Key='customers/transformed_customers.csv',
        Body=csv_buffer
    )
    
    response = s3_mock.get_object(
        Bucket='etl-staging-bucket',
        Key='customers/transformed_customers.csv'
    )
    loaded_data = pd.read_csv(response['Body'])
    
    assert len(loaded_data) == len(transformed_data)
    print(f"✓ Loaded {len(loaded_data)} records to S3")
    
    assert loaded_data['customer_tier'].isin(['bronze', 'silver', 'gold']).all()
    assert loaded_data['has_email'].dtype == bool
    
    print("✅ End-to-end pipeline test PASSED")

# src/lakehouse/bronze/ingest_raw_data.py
from pyspark.sql import SparkSession
from pyspark.sql import functions as F

class BronzeIngestion:
    def __init__(self, spark: SparkSession):
        self.spark = spark
    
    def ingest_orders(self, source_data, bronze_path):
        """Ingest raw orders into Bronze layer."""
        df = self.spark.createDataFrame(source_data)
        
        df = df.withColumn("ingestion_timestamp", F.current_timestamp()) \
               .withColumn("ingestion_date", F.current_date()) \
               .withColumn("file_name", F.lit("source_file.json"))
        
        df.write.format("delta").mode("append").save(bronze_path)
        
        return df

def test_bronze_ingestion_adds_metadata(spark, sample_raw_orders, delta_paths):
    """Test that Bronze layer adds metadata columns."""
    ingestion = BronzeIngestion(spark)
    df = ingestion.ingest_orders(sample_raw_orders, delta_paths['bronze'])
    
    bronze_df = spark.read.format("delta").load(delta_paths['bronze'])
    
    assert 'ingestion_timestamp' in bronze_df.columns
    assert 'ingestion_date' in bronze_df.columns
    assert 'file_name' in bronze_df.columns
    
    assert bronze_df.count() == 5

def test_bronze_ingestion_preserves_raw_data(spark, sample_raw_orders, delta_paths):
    """Test that Bronze layer preserves all raw data including invalid."""
    ingestion = BronzeIngestion(spark)
    ingestion.ingest_orders(sample_raw_orders, delta_paths['bronze'])
    
    bronze_df = spark.read.format("delta").load(delta_paths['bronze'])
    
    invalid_row = bronze_df.filter(F.col("amount") == "invalid").count()
    assert invalid_row == 1
    
    duplicate_orders = bronze_df.filter(F.col("order_id") == "ORD003").count()
    assert duplicate_orders == 2

Step 3: Silver Layer - Data Cleansing

# src/lakehouse/silver/cleanse_data.py
from pyspark.sql import functions as F
from pyspark.sql.types import DecimalType, DateType

class SilverTransformation:
    def __init__(self, spark):
        self.spark = spark
    
    def cleanse_orders(self, bronze_path, silver_path):
        """Transform Bronze to Silver with cleansing logic."""
        df = self.spark.read.format("delta").load(bronze_path)
        
        df = df.dropDuplicates(["order_id"])
        
        df = df.withColumn("amount_clean", 
                          F.when(F.col("amount").cast(DecimalType(10,2)).isNotNull(),
                                 F.col("amount").cast(DecimalType(10,2)))
                          .otherwise(None))
        
        df = df.withColumn("order_date_clean", 
                          F.to_date(F.col("order_date")))
        
        df = df.filter(
            (F.col("customer_id").isNotNull()) &
            (F.col("amount_clean").isNotNull()) &
            (F.col("order_date_clean").isNotNull())
        )
        
        df = df.withColumn("is_valid", F.lit(True)) \
               .withColumn("cleansed_timestamp", F.current_timestamp())
        
        df = df.select(
            F.col("order_id"),
            F.col("customer_id"),
            F.col("amount_clean").alias("amount"),
            F.col("order_date_clean").alias("order_date"),
            F.col("status"),
            F.col("source_system"),
            F.col("is_valid"),
            F.col("cleansed_timestamp")
        )
        
        df.write.format("delta").mode("overwrite").save(silver_path)
        
        return df

def test_silver_removes_duplicates(spark, sample_raw_orders, delta_paths):
    """Test that Silver layer removes duplicate order_ids."""
    bronze = BronzeIngestion(spark)
    bronze.ingest_orders(sample_raw_orders, delta_paths['bronze'])
    
    silver = SilverTransformation(spark)
    silver.cleanse_orders(delta_paths['bronze'], delta_paths['silver'])
    
    silver_df = spark.read.format("delta").load(delta_paths['silver'])
    
    unique_orders = silver_df.select("order_id").distinct().count()
    assert unique_orders == silver_df.count()

# src/lakehouse/gold/aggregate_metrics.py
from pyspark.sql import functions as F
from pyspark.sql.window import Window

class GoldAggregation:
    def __init__(self, spark):
        self.spark = spark
    
    def create_customer_summary(self, silver_path, gold_path):
        """Create Gold layer customer summary metrics."""
        df = self.spark.read.format("delta").load(silver_path)
        
        summary = df.groupBy("customer_id").agg(
            F.count("order_id").alias("total_orders"),
            F.sum("amount").alias("total_revenue"),
            F.avg("amount").alias("avg_order_value"),
            F.min("order_date").alias("first_order_date"),
            F.max("order_date").alias("last_order_date"),
            F.countDistinct("source_system").alias("unique_channels")
        )
        
        summary = summary.withColumn(
            "customer_lifetime_days",
            F.datediff(F.col("last_order_date"), F.col("first_order_date"))
        )
        
        summary = summary.withColumn(
            "customer_segment",
            F.when(F.col("total_orders") >= 5, "loyal")
            .when(F.col("total_orders") >= 2, "repeat")
            .otherwise("new")
        )
        
        summary = summary.withColumn("report_date", F.current_date())
        
        summary.write.format("delta").mode("overwrite").save(gold_path)
        
        return summary

def test_complete_medallion_pipeline(spark, sample_raw_orders, delta_paths):
    """Test complete Bronze → Silver → Gold pipeline."""
    
    print("\n📥 BRONZE: Ingesting raw data...")
    bronze = BronzeIngestion(spark)
    bronze.ingest_orders(sample_raw_orders, delta_paths['bronze'])
    
    bronze_df = spark.read.format("delta").load(delta_paths['bronze'])
    bronze_count = bronze_df.count()
    
    assert bronze_count == 5, f"Expected 5 Bronze records, got {bronze_count}"
    print(f"✓ Bronze layer: {bronze_count} raw records ingested")
    
    print("\n🧹 SILVER: Cleansing and deduplicating...")
    silver = SilverTransformation(spark)
    silver.cleanse_orders(delta_paths['bronze'], delta_paths['silver'])
    
    silver_df = spark.read.format("delta").load(delta_paths['silver'])
    silver_count = silver_df.count()
    
    assert silver_count == 2, f"Expected 2 valid Silver records, got {silver_count}"
    
    assert silver_df.filter(F.col("amount").isNull()).count() == 0
    assert silver_df.filter(F.col("customer_id").isNull()).count() == 0
    assert silver_df.select("order_id").distinct().count() == silver_count
    
    print(f"✓ Silver layer: {silver_count} cleansed records")
    print(f"  - Removed {bronze_count - silver_count} invalid/duplicate records")
    
    print("\n📊 GOLD: Creating business aggregations...")
    gold = GoldAggregation(spark)
    gold.create_customer_summary(delta_paths['silver'], delta_paths['gold'])
    
    gold_df = spark.read.format("delta").load(delta_paths['gold'])
    gold_count = gold_df.count()
    
    assert gold_count > 0, "Gold layer has no records"
    
    expected_cols = ['customer_id', 'total_orders', 'total_revenue', 
                     'avg_order_value', 'customer_segment']
    for col in expected_cols:
        assert col in gold_df.columns, f"Missing column: {col}"
    
    print(f"✓ Gold layer: {gold_count} customer summaries created")
    
    print("\n✅ BUSINESS VALIDATION:")
    
    silver_total = silver_df.agg(F.sum("amount")).first()[0]
    gold_total = gold_df.agg(F.sum("total_revenue")).first()[0]
    
    assert float(silver_total) == pytest.approx(float(gold_total), 0.01), \
        "Revenue mismatch between Silver and Gold"
    
    print(f"  - Revenue consistency: ${silver_total:.2f} (Silver) = ${gold_total:.2f} (Gold)")
    
    segments = gold_df.groupBy("customer_segment").count().collect()
    print(f"  - Customer segments: {dict((r.customer_segment, r['count']) for r in segments)}")
    
    print("\n🎉 MEDALLION PIPELINE TEST PASSED!\n")

def test_medallion_data_lineage(spark, sample_raw_orders, delta_paths):
    """Test data lineage tracking through layers."""
    bronze = BronzeIngestion(spark)
    bronze.ingest_orders(sample_raw_orders, delta_paths['bronze'])

    silver.cleanse_orders(delta_paths['bronze'], delta_paths['silver'])

    filtered_orders = bronze_orders - silver_orders
    print(f"Filtered orders: {filtered_orders}")
    
    assert "ORD004" in filtered_orders  # Null customer_id

2. Naming Conventions

  • Test files: test_*.py or *_test.py
  • Test functions: test_* (descriptive names)
  • Test classes: Test* (optional grouping)
  • Fixtures: Descriptive names indicating what they provide
# Good naming examples

    pass

    pass

@pytest.fixture

    pass

3. Test Independence

Each test should be independent and not rely on the execution order or state from other tests.

# Bad: Tests depend on each other
def test_create_user():
    global user_id
    user_id = create_user("Alice")

def test_update_user():
    update_user(user_id, name="Bob")  # Depends on previous test

# Good: Each test is independent
@pytest.fixture
def user():
    user_id = create_user("Alice")
    yield user_id
    delete_user(user_id)

def test_update_user(user):
    update_user(user, name="Bob")
    assert get_user(user).name == "Bob"

4. Parametrized Tests

Test multiple scenarios with a single test function using parametrization.
